# Design Direction

## Visual Thesis

A late-night project war-room for AP project managers: dark glass, warm working paper, AP red as authority, and teal signal lines as the AI layer connecting scattered project context.

## Content Plan

1. Hero: reposition prompt engineering as clear PM briefing.
2. Agenda: preview the 3 core topics in learning order: good prompts, Claude type, and model choice.
3. Insight: anchor the session with Piyalitt Ittichaiwong's prompt-as-brief idea.
4. PM reality: show context overload across project artifacts.
5. Claude work types: separate Chat, Cowork, and Code by job type.
6. Model map: explain official model trade-offs with a practical recommendation.
7. Prompt practices: job, context, source material, answer format, and review criteria.
8. PM matrix: map AP PM tasks to model and tool choices.
9. Tool deep dive: Chat, Cowork, and Code use cases.
10. Close: prompt takeaway focused on clear job, context, and usable result.

## Interaction Thesis

- Scroll sections behave like presentation slides with keyboard navigation.
- A progress bar and dot navigation keep the presenter oriented.
- Speaker notes are available in-app via a side panel and in a separate markdown file.

## Visual Rules

- Use AP assets sparingly as identity anchors, not corporate slide decoration.
- Avoid generic chatbot bubbles, robot mascots, and SaaS dashboard mosaics.
- Keep sections large, editorial, and readable from a meeting room screen.
- Use repeated panels only for comparison, matrix, and workflow objects.
- Keep mobile usable for review, while prioritizing desktop presentation.

## Palette

- Near-black workspace: `#070b0d`
- Paper: `#f7f0e5`
- AP red: `#e1282e`
- AI signal teal: `#2ee4d3`

## Typography

- Thai/body: `Noto Sans Thai`
- Thai display/headings: `Noto Serif Thai`
- Code/model labels: `Space Mono`
- AP local fonts are included as a fallback identity layer.
