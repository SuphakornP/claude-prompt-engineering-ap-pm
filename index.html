โครงสร้าง Prompt

Prompt ที่ดีสำหรับ PM มี 6 ส่วน ไม่ใช่แค่คำสั่งเดียว

บทบาท เป้าหมาย บริบท ข้อมูลอ้างอิง รูปแบบคำตอบ เกณฑ์ตรวจ 
<role>คุณคือ PM advisor ของ AP Thailand</role>
<goal>สรุป issue log เป็น decision memo สำหรับประชุมผู้บริหาร</goal>
<context>โครงการคอนโด high-rise, delay เกิดจาก vendor drawing</context>
<sources>แนบ minutes, issue log, timeline, vendor reply</sources>
<format>ทำเป็น memo 1 หน้า: สถานการณ์, ทางเลือก, ความเสี่ยง, คำแนะนำ</format>
<check>ก่อนตอบ ให้ตรวจว่าคำแนะนำอ้างอิงแหล่งข้อมูลไหนบ้าง</check>

อ้างอิง official: Prompting best practices

บอกทีมว่า XML tags ไม่ได้มีไว้ให้ดูยาก แต่มีไว้แยกชิ้นส่วนของ brief ให้ Claude อ่านไม่ปนกัน ให้เทียบกับ email brief ที่มีหัวข้อชัดเจน ถ้าคำสั่ง บริบท และข้อมูลนำเข้าอยู่รวมกันเป็นย่อหน้าเดียว โอกาสตีความผิดจะสูงขึ้น

เอกสารยาว

เอกสารยาวให้วางข้อมูลต้นทางก่อน แล้วค่อยถามตอนท้าย

Anthropic แนะนำให้วางเอกสารยาวไว้ด้านบนของ prompt และให้คำถามอยู่ท้ายสุด โดยเฉพาะงานที่มีข้อมูลหลายชุด

01 เอกสารต้นทาง
02 ชื่อไฟล์ + ที่มาข้อมูล
03 คำสั่งงาน
04 รูปแบบคำตอบ
05 คำถามสุดท้าย

Anthropic ระบุว่า สำหรับงานหลายเอกสาร การวางคำถามไว้ท้าย prompt ช่วยให้ผลลัพธ์ดีขึ้นในชุดทดสอบของเขา อ่านที่มา

ตัวอย่าง PM: ถ้าจะให้ Claude สรุป vendor proposal เทียบ requirement อย่าเริ่มด้วยถามก่อนแล้วค่อย paste เอกสารยาว ๆ ให้เริ่มจากแยกเอกสารพร้อมชื่อแหล่งข้อมูล แล้วค่อยบอกงานท้ายสุด และขอให้ยกหลักฐานที่เกี่ยวข้องก่อนสรุป

ตัวอย่างช่วยคุมรูปแบบ

ให้ Claude ดูงานตัวอย่าง ก่อนให้เขียนประเด็นใหม่

ไม่ให้ตัวอย่าง

ช่วยเขียนสรุปสถานะให้ผู้บริหารจากรายการประเด็นนี้

Claude ต้องเดาว่าควรเขียนหัวข้ออะไร ยาวแค่ไหน และต้องมีเรื่องให้ตัดสินใจไหม

ให้ตัวอย่าง + ประเด็นใหม่

ตัวอย่างงานที่ดี

สถานะ: เหลือง | ประเด็น: Vendor drawing ยังไม่ปิด | ผลกระทบ: mock-up เสี่ยงเลื่อน 5 วัน | ต้องตัดสินใจ: อนุมัติ overtime review | เจ้าของ: PM + Design

ข้อมูลประเด็นใหม่

Site access ยังไม่ confirm | กระทบงาน MEP 3 วัน | เจ้าของ: Site PM

Claude จะใช้โครงเดิมกับประเด็นใหม่ ไม่ต้องเดาเอง
1. แปะตัวอย่างงานที่ดี 2. บอก “ใช้รูปแบบเดียวกัน” 3. ใส่ข้อมูลประเด็นใหม่ 4. ขอให้เช็กว่าครบประเด็น เจ้าของ และเรื่องตัดสินใจ

Anthropic แนะนำให้ใช้ตัวอย่างที่เกี่ยวข้อง หลากหลาย และมีโครงสร้าง โดย 3-5 ตัวอย่างมักให้ผลดีที่สุด อ่านที่มา

อธิบายว่า few-shot คือการให้ Claude เห็นตัวอย่างงานที่ดี ก่อนให้เขาทำงานใหม่ตามรูปแบบเดิม ตัวอย่างควรมีโครงที่ชัด เช่น สถานะ ประเด็น ผลกระทบ เรื่องที่ต้องตัดสินใจ และเจ้าของงาน ถ้ามีหลายสถานะให้ใส่ตัวอย่างเขียว เหลือง แดง เพื่อให้ Claude เห็น pattern ครบขึ้น

ถ้าอยากได้ชิ้นงาน ต้องสั่งให้ทำชิ้นงาน
